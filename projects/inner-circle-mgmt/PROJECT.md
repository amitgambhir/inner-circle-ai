# PROJECT.md — Inner Circle AI Self-Management

## Overview

This is the starter use case: the inner-circle-ai agent team managing its own repo. The framework eats its own dogfood.

**Goal:** Maintain, grow, and improve the inner-circle-ai open-source framework using the five agents defined within it.

**Why this use case:** It's self-referential, immediately demonstrable, and every developer who forks the repo can see the framework in action from day one.

---

## Scope

### What's In Scope
- Monitoring and triaging GitHub issues and PRs on the inner-circle-ai repo
- Reviewing community contributions
- Writing release notes for new versions
- Maintaining all documentation (README, guides, SOUL files)
- Tracking repo health metrics (stars, forks, contributors, issue velocity)
- Scanning the AI agent ecosystem for trends and competitor activity
- Creating social media content for milestones and releases

### What's Out of Scope
- Building new features (that's the CEO's direction, not autonomous agent work)
- Making breaking changes to the framework structure without explicit CEO approval
- Engaging with users on behalf of the CEO (agents draft — CEO publishes)

---

## Agent Assignments

| Agent | Focus Area |
|-------|-----------|
| **Curie** | Research ecosystem trends, monitor repo activity, track competitor frameworks |
| **Tesla** | Triage issues, review PRs, write ADRs, maintain technical integrity |
| **Ogilvy** | Release notes, community updates, social posts for milestones |
| **Nightingale** | Documentation quality, repo health metrics, contributor experience |
| **Ada** | Coordinate all of the above, deliver CEO briefings |

---

## Key Metrics (Nightingale Tracks These)

- Stars / forks growth rate
- Issue close time (target: <7 days for bugs, <14 days for features)
- PR merge time (target: <5 days)
- New contributor rate
- Documentation coverage (are all features documented?)
- Community sentiment (GitHub Discussions tone, issue quality)

---

## Competitor Landscape (Curie Monitors These)

- CrewAI — role-playing agent orchestration
- MetaGPT — multi-agent meta programming
- AutoGen / AG2 — conversational agent collaboration
- OpenClaw — always-on agent scheduling
- Swarm (OpenAI) — lightweight multi-agent patterns
- PraisonAI — low-code multi-agent framework

**What to track:** New releases, feature parity, community size, differentiation points.

---

## Constraints

- All content must go through the approval queue (Ada → CEO)
- No direct publishing to any platform without CEO sign-off
- Documentation updates for typos/formatting have standing permission (see CEO.md)
- This is an MIT-licensed open-source project — all output should be shareable

---

## Success Criteria

1. The repo is well-maintained: issues are triaged within 48 hours, PRs reviewed within 5 days
2. Documentation is always current: no outdated instructions, no broken links
3. Community feels heard: every issue gets at least a triage response
4. CEO spends <15 minutes/day on approvals via Ada's briefings
5. The framework demonstrates its own value by managing itself effectively
